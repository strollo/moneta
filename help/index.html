Introduction

Main Features

  1. Open source
  2. Window based GUI
  3. Easy installation: no setup required
  4. Works on local apache server so data is locally maintained
  5. Tracks multiple activities: incomes/outgoings/transfers/credit card payments/loans/debts
  6. Chart based reports including timeseries and multiple format exports (png/pdf,jpeg...)
  7. Tag on activities
  8. Flexible grouping on activities reports on date/tag/source....
  9. Support for multiple users, projects and permissions


Requirements

  • Apache Web Server
  • MySql DBMS

Main concepts

Accounts

Accounts are grouped into categories:

  • Bank and Cash Accounts: Are the default accounts (Bank accounts, Wallet, Debt Cards other investments...) having no reconciliations.
    So they can be a deposit accounts or savings accounts while credit accounts are modeled in the Credit Cards type since they need a reconciliation to a Bank Accounts. Typical operations among these accounts are: withdrawals and bank transfers.
  • Credit Cards: A credit card is a payment card issued to users as a system of payment. It allows the cardholder to pay for goods and services based on the holder's promise to pay for them. The issuer of the card creates a revolving account and grants a line of credit to the consumer (or the user) from which the user can borrow money for payment to a merchant or as a cash advance to the user.
    A credit card is different from a charge card: a charge card requires the balance to be paid in full each month. In contrast, credit cards allow the consumers a continuing balance of debt, subject to interest being charged. A credit card also differs from a cash card, which can be used like currency by the owner of the card. A credit card differs from a charge card also in that a credit card typically involves a third-party entity that pays the seller and is reimbursed by the buyer, whereas a charge card simply defers payment by the buyer until a later date.
  • Incoming Accounts: here you store your entries (salary, incoming gifts and so on).
  • Outgoing Accounts: represents the flows of exits. It is enought to have a single outgoing account (e.g. Expenses). But if you want to distinguish your outgoings you can create different accounts in this category so you will have distinct reports according to each of them.
  • Persons: debts and loans to persons are modeled by storing transactions between bank or cash accounts to / from persons.

Actions

  • New Account allows to create new accounts in the current project. It simply required the account name (e.g. mybank), the type (e.g. Bank account) and the initial balance if any (the current balance amount).
  • New Operation opens a form to track a new activity (expense, income, transaction...). Activities are always represented in the form of money flows from an account to another. For example the money withdraw is represented in Figure 2.
    The system will automatically propose the list of accounts compatible with the activity type chosen (for example the money transfer is not allowed on credit cards so they will be not shown in the target/source fields).
  • Reconciliations are actions for pending transactions like credit cards. The operation on credit cards is made persistent only once it is charged to the ahold bank or credit card statement. Only after this action the money is really considered in the expenses.
  • Manage Tags allows to add new or rename existing tags.
    Notice that new tags are automatically created if inserted in the New Operation form and are not present in the database.


Projects

It is possible to manage different ledgers that will group separated accounts and activities. In addition to each project it is possible to assign a set of users enabled to manage it.



Other features

  • Charts
    Reports are represented with graphical charts that can be exported and locally saved in several formats (png, pdf, gif...).
  • Backup/Restore
    The whole project data can be locally backed up and to keep you data safe.
  • Grouping
    By clicking on a column (e.g. Tag) of the activities view (Figure 3) it is possible to group the activites by the chosen field.
  • Search on description
    On the activities view (Figure 3) it is possible to input a text to search so that only the activities containing that string will be shown.
  • Themes
    The application supports also theming with accessibility features.

Installation

Installation instruction (Ubuntu/Debian)

- Install Apache HTTP Server 

sudo apt-get install apache2 apache2.2-bin apache2-utils apache2.2-common libapache2-mod-php5

- Install MySQL DB 

sudo apt-get install mysql-server-5.5 php5-mysql

- Install Moneta

sudo su
cd /var/www
git clone https://github.com/strollo/moneta.git

- Now you will have a folder /var/www/moneta created.

- Configure the database access

cp /var/www/moneta/moneta.ini.sample /var/www/moneta/moneta.ini
gedit /var/www/moneta/moneta.ini

- You'll find a text like this: 

[database]
db_type = "mysql"
db_hostname = "127.0.0.1"
db_port = "3306"
db_name = "moneta"
db_username = "user"
db_password = "password"

- Configure the parameters according to your MySQL configuration (basically db_username and db_password should be enough).

- Set the proper permissions to run

chown www-data.www-data -R /var/www/moneta

- Restart apache

/etc/init.d/apache2 restart

- Done! Connect to http://127.0.0.1/moneta/index.php

- Login as admin/admin

- Remember to manage users (change the admin password)... And enjoy
